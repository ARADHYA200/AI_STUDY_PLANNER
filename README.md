
# AI Study Planner

This repository contains the AI Study Planner — a full-stack application that generates personalized study schedules, tracks study sessions, and provides analytics to help learners meet their goals.

This single README consolidates previous documentation (BACKEND_API_DOCUMENTATION.md, COMPLETION_STATUS.md, FINAL_SUMMARY.md, INDEX.md, INTEGRATION_GUIDE.md, PROJECT_COMPLETE.md, QUICK_START.md, SETUP_GUIDE.md). Those files have been merged into this document and removed to keep the repo tidy.

---

## Quick summary

- Project: AI Study Planner
- Purpose: AI‑powered schedule generation, progress tracking, analytics
- Components: Backend (Node/Express/MongoDB) and Frontend (React + Vite + Tailwind)

---

## Key Features

- User authentication (register / login) with JWT
- Create/manage subjects and topic breakdowns
- AI-driven schedule generation (weighted priority algorithm)
- Calendar/schedule editor and session logging
- Progress tracking and analytics (streaks, trends, topic progress)
- Responsive React frontend with Tailwind CSS

---

## Tech Stack

- Backend: Node.js, Express, MongoDB, Mongoose
- Frontend: React, Vite, Tailwind CSS, Framer Motion
- Utilities: Axios, react-router, react-toastify, recharts, react-calendar

## Quick Start (development)

1. Backend

```bash
cd backend
npm install
cp .env.example .env   # edit .env (MONGODB_URI, JWT_SECRET)
npm run dev
```

2. Frontend (separate terminal)

```bash
cd frontend
npm install
npm run dev
```

Open: http://localhost:5173

## API Endpoints (summary)

All backend endpoints are mounted under `/api`.

Authentication
- POST /api/auth/register
- POST /api/auth/login
- GET  /api/auth/profile
- PUT  /api/auth/preferences

Subjects
- POST   /api/subjects
- GET    /api/subjects
- GET    /api/subjects/:id
- PUT    /api/subjects/:id
- DELETE /api/subjects/:id
- POST   /api/subjects/:id/topics

Schedules
- POST   /api/schedules/generate
- GET    /api/schedules
- PUT    /api/schedules/:id
- PUT    /api/schedules/:id/complete
- DELETE /api/schedules/:id
- GET    /api/schedules/stats

Progress & Analytics
- POST /api/progress/session
- GET  /api/progress/history
- GET  /api/progress/topic/:topicId
- GET  /api/progress/subject/:subjectId
- GET  /api/progress/analytics/daily
- GET  /api/progress/streak
- GET  /api/progress/trend

Health
- GET /api/health

For request/response examples and full API reference, see the Backend section below.

## Setup details (consolidated)

- Copy `.env.example` to `.env` in `backend/` and provide `MONGODB_URI` and `JWT_SECRET`.
- Start backend with `npm run dev` (uses nodemon).
- Configure frontend `VITE_API_URL` in `frontend/.env` if different from `http://localhost:5000/api`.

If you need the step-by-step setup originally in SETUP_GUIDE, it was merged here — run the quick start steps above for development.

## Development notes

- Use Postman or curl to test endpoints; sample cURL commands are available in the previous API docs (now consolidated).
- Frontend stores token in localStorage and sends Authorization: Bearer <token> header.
- Scheduling algorithm weights: exam proximity (40%), difficulty (35%), user priority (25%).
